## 4. API 서버 처리 플로우 (/api/analyze)

```
┌──────────────────┐
│ 요청 수신         │
│ POST /api/analyze│
└────────┬─────────┘
         │
         ▼
┌──────────────────┐     ┌────────────────────┐
│ HTTP Method 확인  │────▶│ POST 아님 → 405    │
└────────┬─────────┘     └────────────────────┘
         │ POST
         ▼
┌──────────────────┐     ┌────────────────────┐
│ Body 파싱         │────▶│ resume 없음 → 400  │
│ resume 존재?      │     │ "이력서 필수"       │
└────────┬─────────┘     └────────────────────┘
         │ 있음
         ▼
┌──────────────────┐     ┌────────────────────┐
│ 길이 검증         │────▶│ 50,000자 초과 → 400│
│ ≤ 50,000자?      │     └────────────────────┘
└────────┬─────────┘
         │ 통과
         ▼
┌──────────────────┐
│ GEMINI_API_KEY   │     ┌────────────────────┐
│ 존재?            │────▶│ 없음 → 500         │

         │ 있음           └────────────────────┘
         ▼
┌──────────────────┐
│ 프롬프트 구성     │

         │
         ▼
┌──────────────────┐
│ Gemini API 호출   │
│ generateContent()│

         │
         ▼

         │ 유효
         ▼
┌──────────────────┐
```
